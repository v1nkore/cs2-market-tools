# SteamDT — мониторинг продаж скинов CS2

Два трекера показателя **Sold Calculation Today** (продажи за день) с **steamdt.com**:
- **Наклейки команд** (вариант **Holo**) турниров: Stockholm 2021, Antwerp 2022, Copenhagen 2024,
  Shanghai 2024, Austin 2025, **Cologne 2026**. Автографы игроков и стикеры организаторов не отслеживаются.
- **Пушки** (винтовки/пистолеты/ПП/тяжёлое) в качестве **FN/MW**, обычные, со средними продажами **>20/день**.

Результат — наглядные HTML-отчёты с историей по дням.

---

## Как пользоваться — Панель управления

1. Дважды кликните **`Открыть панель.bat`** — откроется окно консоли и страница панели
   в браузере (`http://localhost:4317`). **Окно консоли не закрывайте**, пока работаете.
2. На панели для каждого трекера (Наклейки / Пушки) есть:
   - **▶ Запустить / продолжить** — запускает сбор за сегодня. Если ранее был сбой —
     **продолжит с места остановки** (уже собранное не пересобирается). При сбое браузера
     панель **сама перезапустит** сбор и доведёт день до конца.
   - **■ Остановить** — прервать сбор.
   - **Прогресс-бар** — собрано / всего и проценты.
   - **Открыть отчёт ↗** — открыть HTML-отчёт.
3. Заходите утром, жмёте «Запустить» для наклеек и пушек — и всё.

> Счётчик «Sold Today» на сайте обнуляется в полночь по Пекину (UTC+8 ≈ 19:00 МСК).
> Запуск утром по МСК фиксирует счётчик текущего дня сайта на этот момент.

Автозапуска по расписанию нет — сбор запускается вручную через панель.

### Пушки: разовый отбор (уже выполнен)
Набор пушек (`config_guns.json`) формируется один раз — **`Пушки — отбор скинов (1 раз).bat`**
(отбирает скины со средними продажами >20/день). Менять порог — в `guns_select.mjs` (THRESHOLD),
затем `node guns_select.mjs --finalize`.

## Отчёты (папка `reports`)

| Файл | Что показывает |
|------|----------------|
| **index.html** | Дашборд: сводка, список выгрузок по дням, разбивка по турнирам |
| **history.html** | Сводная матрица: строки — наклейки, столбцы — даты, в ячейках Sold Today |
| **day_ГГГГ-ММ-ДД.html** | Снимок за конкретный день: все поля по каждой наклейке |

В каждой таблице работает поиск по команде, фильтр по турниру и сортировка по любому столбцу (клик по заголовку).

### Поля по каждой позиции
Название · Коллекция (турнир) · **Sold Today** · Цена · Цена Steam · Цена Buff · Цена YouPin · Оборот за день · Дата показателя Sold Today · Дата+время выгрузки.

- **Sold Today** = сделки на китайских площадках + Steam за текущий день (главный показатель, как на карточке сайта).
- **Цена** — минимальная рыночная цена среди BUFF / YouPin / C5 (в юанях ¥).

## Сырые данные

Все собранные значения хранятся в **`data\history.json`** — это «сырьё», из которого
строятся HTML-отчёты. Формат — обычный JSON (без Excel), удобно открыть в браузере/блокноте
или использовать программно. Структура:

```
{
  "items": { "<название наклейки>": { team, tournament, ... } },
  "days":  { "2026-06-15": { scrapedAt, data: { "<наклейка>": { soldToday, price, priceSteam, ... } } } }
}
```

Повторный запуск за тот же день обновляет данные этого дня; за новый день — добавляет новый столбец в историю.

## Список отслеживаемых команд

Список задаётся в **`config.json`** и его легко править вручную:
добавить/убрать команду — просто измените массив `teams` нужного турнира и сохраните файл.
Название наклейки строится как `Sticker | <Команда> (Holo) | <Турнир>`.

Если команда указана с ошибкой или у неё нет такой наклейки, она попадёт в
`data\failed_ГГГГ-ММ-ДД.json` — туда стоит заглянуть после первого запуска.

## Что внутри (для разработчика)

- `scrape.mjs` — сбор данных (Node.js + Playwright, headless-браузер).
- `report.mjs` — генерация HTML-отчётов из `data/history.json`.
- `config.json` — турниры и составы команд.
- `data/` — `history.json`, профиль браузера, логи ошибок.
- `reports/` — готовые HTML.

### Почему через браузер, а не напрямую по API
У сайта есть анти-бот защита: «голые» HTTP-запросы блокируются («environment abnormal»).
Запросы, выполненные внутри настоящего браузера, проходят. Поэтому программа открывает
карточку каждой наклейки в headless-Chrome и читает ответ собственного API сайта
(`/api/item/trade/v1/overview/today`).

### Команды
```
node scrape.mjs                      # полный сбор
node scrape.mjs --tournament austin  # только один турнир
node scrape.mjs --limit 5            # первые 5 позиций (тест)
node report.mjs                      # пересобрать HTML из history.json
```

Панель: `ui_server.mjs` (локальный сервер на `http://localhost:4317`) + `Открыть панель.bat`.
Скрипты пушек: `scrape_guns.mjs`, `report_guns.mjs`, `config_guns.json`, данные в `data_guns/`,
отчёты в `reports_guns/`.

### Требования
- Node.js (проверено на v24)
- Браузер Chromium для Playwright лежит в папке проекта **`pw-browsers`** (а не в системной).
  Все `.bat` задают `PLAYWRIGHT_BROWSERS_PATH=...\pw-browsers` перед запуском node.
  Переустановить браузер при необходимости (из папки проекта):
  ```
  set PLAYWRIGHT_BROWSERS_PATH=%cd%\pw-browsers
  npx playwright install chromium
  ```
