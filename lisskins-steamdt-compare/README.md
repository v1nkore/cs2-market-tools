# lisskins-steamdt-compare

Сравнение цен избранных скинов CS2 с [lis-skins.com](https://lis-skins.com) против Steam, BUFF и YouPin (данные через [SteamDT](https://www.steamdt.com)). Пересборка данных — по кнопке, автоматически, без внешних сервисов.

## Файлы

- `index.html` — интерактивный отчёт (поиск, сортировка, подсветка самой дешёвой площадки).
- `comparison.xlsx` — та же выгрузка в Excel с формулами.
- `data.json` — сырые данные последней пересборки.
- `server.mjs` / `scrape.mjs` / `build.mjs` — локальная панель и сборщик.

## Запуск и обновление по кнопке

Оба сайта блокируют прямые HTTP-запросы антиботом, поэтому сборщик работает через настоящий headless-Chromium (Playwright): lis-skins читается из клиентского vue-query кэша страницы, SteamDT — перехватом ответа его собственного API при загрузке страницы поиска.

**Windows:** двойной клик по `Открыть панель.bat` — при первом запуске сам поставит зависимости и браузер, затем откроет `http://localhost:8317`.

**Вручную:**
```
npm install
npx playwright install chromium
node server.mjs        # панель на http://localhost:8317
```

На странице: вставьте ссылку на список избранного lis-skins (`user_list_id=...`), нажмите **«Пересобрать по ссылке»** — сборка идёт ~5-10 минут с прогрессом, по завершении `index.html`, `comparison.xlsx` и `data.json` **перезаписываются на диске** (следующее открытие файла покажет свежие данные), страница перезагружается сама. Если папка — git-репозиторий, изменения автоматически коммитятся и пушатся (отключить: `GIT_AUTOPUSH=0`).

Пересборка из консоли без панели: `node scrape.mjs --url "<ссылка>"`.

**Docker:**
```
docker build -t lisskins-steamdt-compare .
docker run -p 8317:8317 lisskins-steamdt-compare
```
Внутри контейнера файлы перезаписываются в контейнерной ФС; чтобы результат оставался на хосте, примонтируйте папку: `docker run -p 8317:8317 -v "$(pwd):/app" lisskins-steamdt-compare` (git-пуш в контейнере выключен). Учтите: с датацентровых IP антибот SteamDT может срабатывать чаще, чем с домашнего.

Если `index.html` открыт напрямую как файл (`file://`), кнопка работает в запасном режиме — копирует готовый промпт для Claude Code.

## Как это устроено

- Список избранного lis-skins вытаскивается из клиентского состояния страницы (vue-query кэш, ключ `skins`/`userLists`) — интерфейс сайта не фильтрует выдачу по списку из-за бага, поэтому данные читаются напрямую из кэша; пагинация обходится по `meta.last_page`.
- Цены SteamDT приходят из ответа `/api/skin/market/v3/page` в юанях (CNY); курс CNY→RUB берётся из `localStorage.commonDictionary.rates` того же сайта. Прямой `fetch` к этому API сайт отклоняет (ошибка 108), поэтому читается ответ, инициированный самой страницей поиска.
- `build.mjs` подставляет свежие данные в `index.html` (строка `const DATA = ...`, шапка, ссылка списка) и генерирует `comparison.xlsx` (exceljs).
