# lisskins-steamdt-compare

Сравнение цен избранных скинов CS2 с [lis-skins.com](https://lis-skins.com) против Steam, BUFF и YouPin (данные через [SteamDT](https://www.steamdt.com)). Пересборка данных — по кнопке, автоматически, без внешних сервисов.

## Файлы

- `index.html` — интерактивный отчёт (поиск, фильтры по категории и редкости, сортировка, подсветка самой дешёвой площадки).
- `comparison.xlsx` — та же выгрузка в Excel с формулами.
- `data.json` — сырые данные последней пересборки.
- `server.mjs` / `scrape.mjs` / `build.mjs` — локальная панель и сборщик.

## Зависимости

| Что | Зачем | Где взять |
|---|---|---|
| Node.js ≥ 18 + npm | сервер и сборщик | https://nodejs.org |
| Playwright + Chromium (~500 МБ) | обход антибота обоих сайтов | ставится автоматически при `npm install` (postinstall) |
| Docker (опционально) | запуск без установки Node на машину | Docker Desktop / docker engine |

Интернет-доступ к `lis-skins.com`, `app.lis-skins.com` и `steamdt.com` обязателен.

## Запуск локально

**Windows:** двойной клик по `Открыть панель.bat` — при первом запуске сам поставит зависимости и Chromium (в папку `pw-browsers`), затем откроет панель на http://localhost:8317.

**Любая ОС, вручную:**
```
npm install          # Chromium скачается автоматически (postinstall)
node server.mjs      # панель на http://localhost:8317
```

## Запуск в Docker

```
docker build -t lisskins-steamdt-compare .
# Windows cmd:
docker run -d --name lisskins-compare -p 8317:8317 -v "%cd%:/app" lisskins-steamdt-compare
# PowerShell:  -v "${PWD}:/app"      bash/macOS/Linux:  -v "$(pwd):/app"
```

- Том `-v ...:/app` монтирует папку проекта, чтобы пересборка перезаписывала файлы на хосте. Если в примонтированной папке нет `node_modules` (свежий клон) — контейнер сам доустановит зависимости при старте (браузеры уже внутри образа).
- Можно запускать и без тома — тогда файлы обновляются только внутри контейнера (скачать свежие: `http://localhost:8317/comparison.xlsx`, `/data.json`).
- Git-пуш внутри контейнера выключен (`GIT_AUTOPUSH=0`).
- С датацентровых/VPN IP антибот сайтов может срабатывать чаще, чем с домашнего.

## Использование

На странице вставьте ссылку на список избранного lis-skins (`?user_list_id=...`) и нажмите **«Пересобрать по ссылке»**. Сборка ~10 минут с прогрессом; по завершении `index.html`, `comparison.xlsx` и `data.json` перезаписываются на диске, страница перезагружается сама. Если папка — git-репозиторий (запуск не в Docker), изменения автоматически коммитятся и пушатся (отключить: `GIT_AUTOPUSH=0`).

Пересборка из консоли без панели: `node scrape.mjs --url "<ссылка>"`.

Если `index.html` открыт напрямую как файл (`file://`), кнопка работает в запасном режиме — копирует готовый промпт для Claude Code.

## Как это устроено

- Оба сайта блокируют прямые HTTP-запросы антиботом, поэтому сборщик работает через настоящий headless-Chromium (Playwright).
- Список избранного lis-skins: первая страница читается из SSR-состояния (vue-query кэш), дальше — клики по пагинации внутри SPA с перехватом её API-ответов `/api/v2/obtained-skins` (полные перезагрузки документа ловит Cloudflare-заглушка, которая в headless не проходит). Категория и редкость берутся из `tag_manager_data` позиции.
- Цены SteamDT приходят из ответа `/api/skin/market/v3/page` в юанях (CNY); курс CNY→RUB — из `localStorage.commonDictionary.rates` того же сайта. При рейт-лимите сборщик ждёт 65 секунд и повторяет.
- `build.mjs` подставляет свежие данные в `index.html` (строка `const DATA = ...`, шапка, ссылка списка) и генерирует `comparison.xlsx` (exceljs).
