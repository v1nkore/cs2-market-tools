# cs2-market-radar

Дашборд активного рынка CS2 по данным агрегатора [SteamDT](https://www.steamdt.com): что сейчас откупают, где обваливаются лоты, где скачет цена и на что идёт хайп. С фильтрами по категории/редкости/износу, переключением периода (24ч/3д/7д/30д) и индексами аномалий.

## Зачем

Вместо опроса тысяч отдельных карточек берётся готовый **рейтинг** SteamDT (`ranking/v1/page`), где по каждому предмету сразу приходят цена, изменение цены, число лотов на площадках и его изменение, число сделок и оборот — за периоды 1/3/7/30 дней и популярность. Это ~1000 активно торгуемых позиций за один заход и на порядки меньше запросов, чем открывать карточки поштучно (и не упирается в дневной лимит SteamDT).

## Что показывает

- **Таблица** с сортировкой по любому столбцу; переключатель периода меняет колонки Δцена / Δлоты / сделки на выбранный горизонт.
- **Каскадные фильтры**: категория → редкость → износ/финиш → качество. Опции каждого фильтра сужаются под остальные выбранные (выбрал винтовки — в износе только FN/MW/…; выбрал стикеры — Paper/Holo/Foil).
- **Индексы аномалий** (пороги задаются в `index.html`, константа `TH`):
  - **Индекс сделок** = сделки сегодня ÷ средние за 30 дней (напр. среднее 10, сегодня 50 → ×5). Ловит всплески спроса. Порог ≥3×.
  - **Обвал лотов** — резкое падение числа лотов на площадках за день (напр. −70%): скупка/дефицит. Порог ≤−50%.
  - **Скачок цены** — аномальное дневное изменение цены. Порог ≥15%.
  - **Взлёт популярности** — резкий подъём в рейтинге внимания SteamDT (`hotRankChange`). Порог ≥200 мест.
- **KPI-плитки** и подсветка/бейджи аномальных строк; переключатель «только аномалии» и пресеты сортировки.

Данные: цены в юанях (¥, как отдаёт SteamDT). Клик по названию — карточка предмета на SteamDT.

## Зависимости

- Node.js ≥ 18 + npm; Chromium (~500 МБ) ставится сам при `npm install` (postinstall).
- Интернет-доступ к steamdt.com. Сбор идёт через headless-Chromium: прямые запросы к рейтингу сайт подписывает/режет, поэтому листаем внутри самой SPA скроллом (её запросы проходят).

## Запуск локально

**Windows:** двойной клик по `Открыть панель.bat` → http://localhost:8319 (первый запуск ставит зависимости сам).

**Любая ОС:**
```
npm install          # Chromium скачается автоматически
node server.mjs      # панель на http://localhost:8319
```
На панели жмите **«Собрать заново»** — сбор ~3-5 минут с прогрессом; по завершении `index.html` и `data.json` перезаписываются, страница перезагружается. Вне Docker при git-репозитории изменения авто-коммитятся и пушатся (отключить: `GIT_AUTOPUSH=0`).

Собрать из консоли без панели: `node scrape.mjs` (или `--limit N` для теста).

## Запуск в Docker

```
docker build -t cs2-market-radar .
docker run -d --name cs2-radar -p 8319:8319 -v "%cd%:/app" cs2-market-radar
# PowerShell: -v "${PWD}:/app"   bash: -v "$(pwd):/app"
```
Том монтирует папку, чтобы сбор перезаписывал файлы на хосте; при свежем клоне без `node_modules` контейнер доустановит зависимости сам. Git-пуш в контейнере выключен.

## Как это устроено

- `scrape.mjs` — открывает страницу рейтинга, пошагово скроллит окно (список виртуализируется — прыжок в низ не годится), перехватывает ответы `ranking/v1/page`, собирает ~1000 позиций и пишет `data.json` + дневной снимок в `data/history.json`.
- `build.mjs` — подставляет данные в `index.html` (строка `const DATA = …` и шапка).
- `server.mjs` — локальная панель с кнопкой сбора.
- Индексы и фильтры считаются в самом дашборде (JS), пороги — в константе `TH` в `index.html`.
