# cs2-sticker-tracker

Ежедневный трекер продаж **командных Holo-наклеек** CS2 со [SteamDT](https://www.steamdt.com): снимает показатель «продано сегодня» (Sold Today) по наклейкам команд турниров, копит историю по дням и строит HTML-отчёты.

Отслеживаются варианты **Holo** наклеек команд турниров: Stockholm 2021, Antwerp 2022, Rio 2022, Copenhagen 2024, Shanghai 2024, Austin 2025, Cologne 2026. Автографы игроков и стикеры организаторов не отслеживаются. Список команд задаётся в `config.json`.

## Что показывает

Отчёты в папке `reports/`:

| Файл | Что показывает |
|------|----------------|
| **index.html** | Дашборд: сводка, список выгрузок по дням, разбивка по турнирам |
| **history.html** | Сводная матрица: строки — наклейки, столбцы — даты, в ячейках Sold Today |
| **day_ГГГГ-ММ-ДД.html** | Снимок за конкретный день: все поля по каждой наклейке |

По каждой позиции: название · турнир · **Sold Today** · цена (мин. среди BUFF/YouPin/C5, ¥) · цена Steam/Buff/YouPin · оборот за день · даты. В таблицах — поиск, фильтр по турниру и сортировка по столбцам.

## Зависимости

- Node.js ≥ 18 + npm; Chromium (~500 МБ) ставится сам при `npm ci` (postinstall).
- Интернет-доступ к steamdt.com. Сбор идёт через headless-Chromium: прямые запросы к API SteamDT режет анти-бот, поэтому данные читаются со страниц карточек, которые сайт отдаёт сам.
- Путь к браузерам — переменная `PLAYWRIGHT_BROWSERS_PATH` (в Docker-образе `/ms-playwright`), иначе локальная папка `pw-browsers`.

## Запуск локально

**Windows:** двойной клик по `Открыть панель.bat` — при первом запуске сам ставит зависимости и Chromium, затем открывает панель на http://localhost:4317.

**Любая ОС:**
```
npm ci                # Chromium скачается автоматически
node ui_server.mjs    # панель на http://localhost:4317
```

На панели одна кнопка **«Запустить / продолжить»** — собирает сегодняшний день; при сбое продолжает с места остановки, недобранное добирает авто-возобновлением. По завершении отчёты в `reports/` перестраиваются; «Открыть отчёт» открывает дашборд. «Собрать заново» перевыгружает сегодняшний день с нуля.

Из консоли без панели:
```
node scrape.mjs                      # полный сбор за сегодня
node scrape.mjs --tournament austin  # только один турнир
node scrape.mjs --limit 5            # первые 5 позиций (тест)
node report.mjs                      # пересобрать HTML из data/history.json
```

## Запуск в Docker

```
docker build -t cs2-sticker-tracker .
docker run -p 4317:4317 -v "%cd%/data:/app/data" -v "%cd%/reports:/app/reports" cs2-sticker-tracker
# PowerShell: ${PWD} вместо %cd%   bash: $(pwd)
```
Тома для `data/` и `reports/` — чтобы история и отчёты не терялись между перезапусками. Кнопка «Открыть отчёт» в контейнере не сработает (нет браузера на хосте) — открывайте `reports/index.html` из примонтированной папки.

## Данные и история

Собранные значения хранятся в `data/history.json` (сырьё для отчётов), формат:
```
{
  "items": { "<название наклейки>": { team, tournament, ... } },
  "days":  { "2026-07-23": { scrapedAt, data: { "<наклейка>": { soldToday, price, priceSteam, ... } } } }
}
```
Повторный запуск за тот же день обновляет данные дня; за новый день добавляет новый столбец. `data/` и `reports/` в git не хранятся (личные данные) — свежий клон начинает историю с нуля.

## Внутри (для разработчика)

- `scrape.mjs` — сбор за день (тонкая обёртка над движком).
- `scrape_engine.mjs` — движок: открывает карточку каждой наклейки, читает «продано сегодня» из ряда `type-trend` и цены из `skin/v1/item`.
- `report.mjs` — генерация HTML-отчётов из `data/history.json`.
- `ui_server.mjs` — панель на `http://localhost:4317`.
- `config.json` — турниры и составы команд.

### Почему через браузер, а не по API
У SteamDT анти-бот: прямые запросы к его ручкам подписываются/режутся. Запросы, выполненные самой страницей карточки, проходят — поэтому движок открывает карточку в headless-Chromium и читает ответы её собственного API. Есть суточный лимит запросов на IP для анонимных пользователей — для более высокого потолка можно войти в аккаунт SteamDT.
